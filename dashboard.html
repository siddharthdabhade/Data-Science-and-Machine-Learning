1. Evaluate Structured & Semi-Structured Dataset

Aim: Load, inspect and compare CSV (structured) vs JSON (semi-structured) data.

Objective / Aim

  • Differentiate between structured and semi-structured data.
  • Load CSV & JSON and normalize JSON to tabular format.

Theory / Concept

Structured data follows a fixed schema (tables). JSON and XML are semi-structured — nested fields that must be flattened for most ML algorithms.

Tools / Software

Python 3.11, Jupyter, pandas, numpy, json

Dataset

sample_sales.csv (500 rows) & data.json (transaction logs)

Procedure / Steps

  1. Import libraries: pandas, json.
  2. Load CSV: pd.read_csv() and inspect head(), info().
  3. Load JSON: json.load(), then pd.json_normalize().
  4. Compare schema, completeness, and nesting levels.
Start → Import libraries → Load CSV → Load JSON → Normalize JSON → Compare → Result

Code / Implementation (snippet)

import pandas as pd
import json

# Structured
df_csv = pd.read_csv('sample_sales.csv')
print(df_csv.head())

# Semi-structured
with open('data.json') as f:
    data = json.load(f)

# normalize nested transactions
from pandas import json_normalize
df_json = json_normalize(data, record_path=['transactions'], meta=['customer_id'])
print(df_json.head())

Output / Observations

CSV: tabular, ready for analysis.
JSON: nested; after normalization, becomes tabular with repeated customer_id per transaction.

Result / Inference

Structured data is straightforward for ML pipelines; semi-structured data provides richer hierarchies but requires preprocessing.

Viva / Review Questions

  1. Define structured and semi-structured data.
  2. Why use pd.json_normalize()?
  3. Give two examples of semi-structured data sources.
